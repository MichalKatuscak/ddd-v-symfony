# DDD Symfony 8 — Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign celého webu (21 stránek) podle design bundle „Architektonický blueprint v tmavém editoru". Texty článků zůstávají; mění se layout, fonty (Inter + JetBrains Mono), barvy (oklch dark IDE), navigace (4 kategorie) a komponenty.

**Architecture:** Vite + Symfony 8 + Twig. CSS rozdělen do 6 modulů (`tokens`, `base`, `chrome`, `article`, `landing`, `hljs-theme`). Komponenty jako Twig partials (callout, code_block, diagram, article_toc, article_head). 6 fází; každá samostatně commit-able. Bez automatizovaných testů (visual verification přes `symfony server:start`).

**Tech Stack:** PHP 8.4, Symfony 8, Twig, Vite 8, highlight.js (existující — pivot ze specu, který říkal Prism), JetBrains Mono + Inter (woff2 subsety), vanilla JS (žádné nové knihovny).

**Spec deviation:** Spec uvádí Prism; projekt už má highlight.js. Místo přidávání další knihovny mapujeme highlight.js token classes na design tokens. Visual výsledek je totožný.

- [ ] **Step 2: Extrahovat URL pro každý subset a stáhnout**

Spustit (v `/tmp/inter-fetch`):
```bash
# Bunny.net subset names: latin, latin-ext, cyrillic, cyrillic-ext, vietnamese
# Soubor pro weight 400 stačí — Inter na bunny obsahuje variable styles ve standalone weight files,
# ale variable woff2 se subset podporou je nejjednodušší: stáhnout všechny URLs z CSS

grep -oE "https://fonts\.bunny\.net/[^)]+\.woff2" inter.css | sort -u > urls.txt
wc -l urls.txt    # očekávaný počet: 20 (5 subsetů × 4 weighty) nebo 5 (variable)

# Stáhnout všechny
mkdir -p downloaded
while IFS= read -r url; do
  fname=$(basename "$url")
  curl -sL "$url" -o "downloaded/$fname"
done < urls.txt
ls -la downloaded/
```

Očekávaný výstup: ~20 woff2 souborů (jeden per (subset × weight) kombinace). Každý ~10–30 KB.

- [ ] **Step 3: Sloučit weighty do jednoho variable souboru per subset přes pyftsubset**

Bunny.net dodává standalone weighty. Spec chce variable font per subset (jeden soubor obsahuje weight range 400–700). Použijeme `pyftsubset` z fonttools k vytvoření subset variable souboru z Inter v4 release na rsms.me:

```bash
# Instalace nástrojů (jednorázově)
pip install --user fonttools brotli

# Stažení Inter v4 variable woff2 ze stable upstream
cd /tmp/inter-fetch
curl -L "https://github.com/rsms/inter/releases/download/v4.0/Inter-4.0.zip" -o inter4.zip
unzip -o inter4.zip 'InterVariable.ttf' -d .

# Subsetting (ENV variables pro unicode-range každého subsetu)
LATIN="U+0000-00FF,U+0131,U+0152-0153,U+02BB-02BC,U+02C6,U+02DA,U+02DC,U+0304,U+0308,U+0329,U+2000-206F,U+20AC,U+2122,U+2191,U+2193,U+2212,U+2215,U+FEFF,U+FFFD"
LATIN_EXT="U+0100-02BA,U+02BD-02C5,U+02C7-02CC,U+02CE-02D7,U+02DD-02FF,U+0304,U+0308,U+0329,U+1D00-1DBF,U+1E00-1E9F,U+1EF2-1EFF,U+2020,U+20A0-20AB,U+20AD-20C0,U+2113,U+2C60-2C7F,U+A720-A7FF"
CYR="U+0301,U+0400-045F,U+0490-0491,U+04B0-04B1,U+2116"
CYR_EXT="U+0460-052F,U+1C80-1C8A,U+20B4,U+2DE0-2DFF,U+A640-A69F,U+FE2E-FE2F"
VIET="U+0102-0103,U+0110-0111,U+0128-0129,U+0168-0169,U+01A0-01A1,U+01AF-01B0,U+0300-0301,U+0303-0304,U+0308-0309,U+0323,U+0329,U+1EA0-1EF9,U+20AB"

mkdir -p subsets
~/.local/bin/pyftsubset InterVariable.ttf --output-file=subsets/inter-latin.woff2 --flavor=woff2 --unicodes="$LATIN" --layout-features='*'
~/.local/bin/pyftsubset InterVariable.ttf --output-file=subsets/inter-latin-ext.woff2 --flavor=woff2 --unicodes="$LATIN_EXT" --layout-features='*'
~/.local/bin/pyftsubset InterVariable.ttf --output-file=subsets/inter-cyrillic.woff2 --flavor=woff2 --unicodes="$CYR" --layout-features='*'
~/.local/bin/pyftsubset InterVariable.ttf --output-file=subsets/inter-cyrillic-ext.woff2 --flavor=woff2 --unicodes="$CYR_EXT" --layout-features='*'
~/.local/bin/pyftsubset InterVariable.ttf --output-file=subsets/inter-vietnamese.woff2 --flavor=woff2 --unicodes="$VIET" --layout-features='*'

ls -la subsets/
```

Očekávaný výstup: 5 woff2 souborů, každý ~5–30 KB (latin nejmenší, latin-ext největší).

### Task 2.2: Vytvořit `topnav.js` — mobile toggle

**Files:**

- [ ] **Step 1: Napsat skript**

Vytvořit `assets/scripts/topnav.js`:

```javascript
// ──────────────────────────────────────────────────────────────────────────
// Topnav — mobile drawer toggle
// ──────────────────────────────────────────────────────────────────────────

document.addEventListener('DOMContentLoaded', function () {
  const toggle = document.querySelector('.nav-toggle');
  const drawer = document.querySelector('.nav-drawer');
  const backdrop = document.querySelector('.nav-drawer-backdrop');

  if (!toggle || !drawer) return;

  function setOpen(open) {
    toggle.setAttribute('aria-expanded', String(open));
    drawer.dataset.open = String(open);
    if (backdrop) backdrop.dataset.open = String(open);
    document.body.style.overflow = open ? 'hidden' : '';
  }

  toggle.addEventListener('click', function () {
    const isOpen = drawer.dataset.open === 'true';
    setOpen(!isOpen);
  });

  if (backdrop) {
    backdrop.addEventListener('click', function () { setOpen(false); });
  }

  // ESC zavře drawer
  document.addEventListener('keydown', function (e) {
    if (e.key === 'Escape' && drawer.dataset.open === 'true') setOpen(false);
  });

  // Klik na link v draweru zavře drawer
  drawer.querySelectorAll('a').forEach(function (link) {
    link.addEventListener('click', function () { setOpen(false); });
  });
});
```

```javascript
// ──────────────────────────────────────────────────────────────────────────
// Article TOC — generuje seznam z <h2> v .art-body, scroll-spy aktivního
// ──────────────────────────────────────────────────────────────────────────

document.addEventListener('DOMContentLoaded', function () {
  const tocList = document.querySelector('[data-toc-list]');
  const body = document.querySelector('.art-body');
  if (!tocList || !body) return;

  const headings = Array.from(body.querySelectorAll('h2'));
  if (headings.length === 0) return;

  // Slugify Czech: lowercase, ASCII fold approx, remove non-alphanum
  function slugify(text) {
    return text
      .normalize('NFD')
      .replace(/[̀-ͯ]/g, '')
      .toLowerCase()
      .replace(/[^a-z0-9]+/g, '-')
      .replace(/^-+|-+$/g, '');
  }

  // Render TOC items
  const chapterEl = document.querySelector('[data-chapter-number]');
  const chapterNum = chapterEl ? chapterEl.dataset.chapterNumber : '';

  const items = headings.map(function (h, i) {
    if (!h.id) h.id = slugify(h.textContent || ('section-' + (i + 1)));
    const num = chapterNum
      ? chapterNum + '.' + String(i + 1).padStart(2, '0')
      : String(i + 1).padStart(2, '0');

    // Pokud h2 obsahuje <span class="h-num">, použij text bez něj
    const numSpan = h.querySelector('.h-num');
    const titleText = numSpan
      ? h.textContent.replace(numSpan.textContent, '').trim()
      : h.textContent.trim();

    const li = document.createElement('li');
    li.dataset.targetId = h.id;
    li.innerHTML =
      '<span class="toc-num">' + num + '</span>' +
      '<a href="#' + h.id + '"><span class="toc-text">' + titleText + '</span></a>';
    return li;
  });

  items.forEach(function (li) { tocList.appendChild(li); });

  // Scroll-spy přes IntersectionObserver
  const linkByHash = new Map();
  items.forEach(function (li) { linkByHash.set(li.dataset.targetId, li); });

  const observer = new IntersectionObserver(function (entries) {
    entries.forEach(function (entry) {
      if (entry.isIntersecting) {
        const li = linkByHash.get(entry.target.id);
        if (!li) return;
        items.forEach(function (x) { x.classList.remove('toc-current'); });
        li.classList.add('toc-current');
      }
    });
  }, {
    rootMargin: '-80px 0px -60% 0px',
    threshold: 0.1
  });

  headings.forEach(function (h) { observer.observe(h); });
});
```

### Task 3.7: Vytvořit `code-block.js` — line wrapping + copy button

**Files:**

Vytvořit `assets/scripts/code-block.js`:

```javascript
// ──────────────────────────────────────────────────────────────────────────
// Code block — po hljs highlight wrapne každý řádek, řeší copy button
// ──────────────────────────────────────────────────────────────────────────

window.__enhanceCodeBlock = function (codeEl) {
  if (codeEl.dataset.enhanced === 'true') return;
  codeEl.dataset.enhanced = 'true';

  const figure = codeEl.closest('figure.code');
  const pre = codeEl.parentElement;
  const hlAttr = pre ? pre.dataset.highlight : '';
  const highlights = (hlAttr || '').split(',').map(function (s) { return parseInt(s.trim(), 10); }).filter(Boolean);

  // Highlight už proběhl (hljs.highlightElement v app.js). Vezmeme innerHTML a rozsekáme po \n.
  const html = codeEl.innerHTML;
  const lines = html.split('\n');
  // Pokud poslední řádek je prázdný (kvůli trailing newline), zahodíme ho
  if (lines.length > 1 && lines[lines.length - 1].replace(/\s/g, '') === '') lines.pop();

  const wrapped = lines.map(function (line, i) {
    const num = i + 1;
    const isHl = highlights.includes(num) ? ' ln-hl' : '';
    const safeLine = line === '' ? '&nbsp;' : line;
    return (
      '<span class="ln' + isHl + '">' +
      '<span class="ln-num">' + num + '</span>' +
      '<span class="ln-text">' + safeLine + '</span>' +
      '</span>'
    );
  }).join('');

  codeEl.innerHTML = wrapped;
};

document.addEventListener('DOMContentLoaded', function () {
  // Copy button delegation
  document.body.addEventListener('click', function (e) {
    const btn = e.target.closest('[data-copy]');
    if (!btn) return;

    const figure = btn.closest('figure.code');
    if (!figure) return;
    const codeEl = figure.querySelector('pre code');
    if (!codeEl) return;

    // Sebrat čistý text (bez line numbers spans) — vezmeme textContent z .ln-text
    const lns = codeEl.querySelectorAll('.ln-text');
    const text = lns.length > 0
      ? Array.from(lns).map(function (el) { return el.textContent; }).join('\n')
      : codeEl.textContent;

    if (navigator.clipboard && navigator.clipboard.writeText) {
      navigator.clipboard.writeText(text).then(function () { showCopied(btn); }).catch(function () {});
    } else {
      // Fallback
      const ta = document.createElement('textarea');
      ta.value = text;
      ta.style.position = 'fixed'; ta.style.opacity = '0';
      document.body.appendChild(ta);
      ta.select();
      try { document.execCommand('copy'); showCopied(btn); } catch (_) {}
      document.body.removeChild(ta);
    }
  });

  function showCopied(btn) {
    const label = btn.querySelector('.code-copy-label');
    if (!label) return;
    const original = label.textContent;
    label.textContent = 'zkopírováno ✓';
    btn.classList.add('code-copied');
    setTimeout(function () {
      label.textContent = original;
      btn.classList.remove('code-copied');
    }, 1400);
  }
});
```

## Phase 5: Migrace 16 článkových šablon

Cíl: přepsat každou článkovou šablonu do nového layoutu s `_partials/article_head`, `_partials/article_toc` a body komponenty (callout, code_block, diagram). Texty zůstávají 1:1.

### Migration Recipe (referenční)

Tato sekce je referenční — každý migration task níže ji aplikuje s vlastními hodnotami.

**Výchozí stav** (typická šablona před migrací):
```twig
{% extends 'base.html.twig' %}
{% block title %}Něco | DDD Symfony{% endblock %}
{% block meta_description %}...{% endblock %}
{% block body %}
<article>
  <h1>...</h1>
  <p>...</p>
  <h2>...</h2>
  <p>...</p>
  <pre><code class="language-php">...</code></pre>
  <div class="warning">...</div>
  <img src="..." alt="...">
</article>
{% endblock %}
```

**Cílový stav** (po migraci):
```twig
{% extends 'base.html.twig' %}

{% block title %}<TITLE> | DDD Symfony{% endblock %}
{% block meta_description %}<DESC>{% endblock %}
{% block breadcrumb_name %}<BREADCRUMB>{% endblock %}

{% block body %}
<article class="article">
    {% include '_partials/article_head.html.twig' with {
        chapter_number: '<NN>',
        category: '<KATEGORIE>',
        title: '<TITLE>',
        deck: '<DECK_HTML>',
        reading_time: <MIN>,
        difficulty: <1-4>,
        last_updated: '2026-04-27'|date_modify('+0 days'),
        author: 'M. Katuščák'
    } %}

    {% include '_partials/article_toc.html.twig' %}

    <div class="art-body" data-chapter-number="<NN>">
        {# === původní obsah článku se sem přepíše s úpravami níže === #}
    </div>
</article>
{% endblock %}
```

**Per-element transformace v `art-body`:**

1. **Vymazat `<h1>`** — title se renderuje v `_partials/article_head` přes `title:` argument.

2. **První `<p>` po `<h1>`** — pokud slouží jako lead/deck, přesuň jeho obsah do argumentu `deck:` v `article_head`. Pokud první `<p>` je už součástí first sekce (ne lede), nech ho a deck nastav z `meta_description`.

3. **`<h2>Něco</h2>`** → `<h2 id="<slug>" class="h-section"><span class="h-num"><NN>.<MM></span> Něco</h2>` kde `<MM>` je pořadí h2 v článku počínaje 01.

4. **`<h3>` a hlubší** — ponechat beze změny (article.css je pokrývá).

5. **`<pre><code class="language-X">...</code></pre>`** → použít `_partials/code_block`. Filename a highlights vyextrahuj z kontextu (komentáře, předchozí věta, název v kódu). Pokud nejsou jasné, dej `filename: 'snippet.X'` a `highlights: []`.

   ```twig
   {% set _code %}<?php
   declare(strict_types=1);
   ...
   {% endset %}
   {% include '_partials/code_block.html.twig' with {
       filename: 'src/Order/Domain/Order.php',
       language: 'php',
       code: _code,
       highlights: []
   } %}
   ```

   **Pozor:** Twig `{% set %}` blok zachovává whitespace. Indent uvnitř `{% set _code %}...{% endset %}` musí být na sloupci 0 (žádný společný indent), jinak se to projeví v code blocku jako extra whitespace. Pokud potřebuješ indent v Twig kódu pro čitelnost, použij `|raw` filter na escaped string místo capture, nebo použij `code: code_str` proměnnou.

6. **`<div class="warning">`, `<div class="tip">`, `<div class="info">` apod.** → `_partials/callout` s typem podle obsahu:
   - „varování / pozor" → `type: 'warn'`
   - „anti-vzor / nepoužívejte" → `type: 'anti'`
   - „doporučený vzor" → `type: 'pattern'`
   - „poznámka / info" → `type: 'note'`

   ```twig
   {% include '_partials/callout.html.twig' with {
       type: 'note',
       label: 'Poznámka',
       body: '<p>...</p>'
   } %}
   ```

7. **`<img src="...diagrams/...">` nebo inline SVG** → `_partials/diagram`:
   ```twig
   {% include '_partials/diagram.html.twig' with {
       fig: '<NN>.<X>-A',
       title: '<DIAGRAM_TITLE>',
       src: 'images/diagrams/...',
       caption: ''
   } %}
   ```

8. **FAQ sekce** (pokud existuje) — řešena samostatně v Phase 6 přes přepsaný `_partials/faq.html.twig`. V tomto kroku ji nech tak, jak je — vrátíme se k ní.

**Verification po migraci jedné stránky:**
- `npx vite build` PASS
- `php bin/console cache:clear`
- `curl -s -o /dev/null -w "%{http_code}\n" http://127.0.0.1:8000/<URL>` → 200
- Visual: title je v breadcrumb, h2 mají `04.01`, `04.02` atd. mono accent prefix, code blocky mají chrome (filename + copy), TOC vlevo se generuje z h2.
